## 5. 鼠标交互逻辑

| 行号范围 | 说明 |
| --- | --- |
| 192-200 | `on_left_press` 先判断是否点到了当前选中点的手柄端点；如果点到了，就直接进入手柄拖动。 |
| 202-214 | 如果不是手柄端点，再判断是否点到了锚点；如果是空白处且绘画模式开启，就新增一个点，但不自动选中。 |
| 216-230 | `on_left_drag` 根据当前拖动模式决定是移动锚点还是拖动手柄。 |
| 232-235 | `on_left_release` 在松开鼠标时结束拖动状态。 |
| 237-247 | `on_right_click` 删除被点击的锚点，并同步修正选中索引。 |
| 249-255 | `add_point` 负责新增点；`select=False` 表示新增后默认不选中。 |
| 257-264 | `undo_point` 撤销最后一个点，并清除拖动状态。 |
| 266-274 | `clear_points` 清空所有点，并清除拖动状态。 |
| 276-278 | `toggle_draw_mode` 目前只负责刷新状态文本，真正的开关行为由 `draw_enabled` 和勾选框直接控制。 |

## 6. 画布重绘

| 行号范围 | 说明 |
| --- | --- |
| 291-305 | 如果点数不少于 2，就先把曲线采样成折线，再按当前颜色绘制到画布上。 |
| 307-313 | 如果当前没有任何点，画布保持空白，不再显示“单击空白处添加第一个点”之类的提示。 |
| 315-318 | 当“显示点”开关开启时，依次绘制所有锚点。 |
| 320 | 如果当前有选中点，就额外绘制它的手柄。 |

## 7. 锚点与手柄绘制

| 行号范围 | 说明 |
| --- | --- |
| 321-331 | `_draw_point` 负责绘制锚点圆点；被选中的点会显示为黑色实心，未选中的点则是白底黑边。 |
| 333-374 | `_draw_handle` 负责绘制左右对称的方向手柄，包括连线、箭头和圆形端点。 |
| 376-386 | `_nearest_point_index` 用于判断鼠标是否点到了某个锚点，并返回最近点的索引。 |
| 388-397 | `_handle_hit` 判断当前选中点的左右手柄是否被点中，作为拖动辅助判断。 |
| 399-412 | `_selected_handle_endpoint` 只对当前选中点生效，返回用户点到的是左手柄还是右手柄。 |
| 414-423 | `_handle_points` 计算左右两个对称手柄的坐标；如果手柄已经被用户手动调整过，就使用保存的偏移量。 |
| 425-432 | `_handle_end` 返回单侧手柄坐标，供曲线控制点计算使用。 |

## 8. 手柄长度与曲线采样

| 行号范围 | 说明 |
| --- | --- |
| 434-456 | `_auto_handle_offset` 根据相邻点计算默认手柄的方向和长度；长度会按比例缩放，并限制在最小值和最大值之间，避免手柄太短或太长。 |
| 458-478 | `_build_curve_polyline` 把所有点之间的曲线段转换为折线，并保证每一段首尾衔接。 |
| 480-504 | `_sample_cubic_bezier` 使用三次贝塞尔公式对每一段进行采样，让曲线看起来更平滑。 |
| 506-510 | `_distance_sq` 计算两点之间的平方距离，用于命中检测，避免频繁开平方。 |
| 512-513 | `_clamp` 把数值限制在给定范围内，防止点或手柄拖出画布。 |
| 515-524 | `_contrast_text_color` 根据背景颜色亮度自动选择黑字或白字，保证颜色预览上的文字可读。 |
